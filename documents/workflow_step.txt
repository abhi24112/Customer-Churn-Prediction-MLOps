1. Create src/components: hold entire ETL, model trainig, saving code

2. Create src/model_config: config.yaml file -> have all the configuration for model , mlflow etc.

3. Create src/pipelines: contains -> 
    a. training_pipline.py : that run the entire components of the ETL, and model training using component folder files

    b. tuning_pipeline.py : This will do the model hyperparameter tuning.

4. Git Initialize : git init (before dvc init)

5. DVC Initialize : dvc init (requires git)

    steps:  git init
            dvc init
            dvc add src/data/raw_data
            git add .
            git commit -m "Initialize project with DVC"
            (step up the git repo with project)
            git push -u origin main

6. DVC (IMPORTANT PART - MAIN SOURCE)
    - ingestion → loading → preprocessing → splitting → training → evaluation
    - Creating `dvc.yaml`
